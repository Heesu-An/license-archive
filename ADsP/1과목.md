**<데이터베이스>**

-   🚨🚨🚨 데이터베이스 특징 (4가지, **공통저변**)
    -   공용 데이터
    -   통합된 데이터: 중복 최소화
    -   저장된 데이터
    -   변화되는 데이터: 새로운 데이터 삽입, 기존 데이터 삭제 및 갱신으로 항상 변화하면서, 현재의 정확한 데이터 유지
-   데이터베이스 설계 절차 (**개논물**)
    -   요구조건 분석 → 개념적 설계 → 논리적 설계 → 물리적 설계
-   🚨 기업 활용 데이터베이스
    -   OLTP: **거래 단위**
    -   OLAP: 정보 위주 분석 처리
    -   CRM: **고객 관계** 관리
    -   SCM: **공급망** 연결
    -   ERP: **경영** 자원
    -   RTE: 빠른 의사결정
    -   BI: **레포트** 지원
    -   BA: 통계 기반 비지니스 통찰력
-   DBMS: 응용프로그램들이 데이터베이스를 공유하며 사용할 수 있는 환경을 제공하는 소프트웨어
-   DBMS(데이터베이스관리시스템) 종류
    -   관계형 DBMS: 테이블 ex. MySQL, Oracle, MS-SQL
    -   객체지향 DBMS: 객체 형태로. 미디어 및 이미지 관리.
    -   NoSQL: 비정형 데이터 ex. **HBase, MongoDB, Cassandra**

**<빅데이터>**

-   단위: PB < EB < ZB < YB(2^80)
-   빅데이터 출현 배경
    -   클라우드 컴퓨팅, 분산 컴퓨팅
    -   모바일 시대, 인터넷 발전, 디지털화의 급진전, 저장 기술의 발전과 가격하락
-   빅데이터 활용을 위한 3대 요소: (**인자기**) 인력, 자원, 기술
-   빅데이터 활용 기본 테크닉
    -   연관규칙 학습, 분류분석(유형분석), 회귀분석, 기계학습, 감정분석, 소셜 네트워크 분석, **유전 알고리즘(최적화)**
-   빅데이터 기능: **렌즈**, **플랫폼**
-   빅데이터 가치 패러다임의 변화(**디지 털카**) : 디지털화 → 연결 → 에이전시
-   빅데이터 특징 (3가지)
    -   3V(Volume, Variety, Velocity) + 4V(Value)
-   빅데이터 시대 위험과 통제
    -   개인정보 동의제보다 책임제 강화해야 한다.
    -   알고리즈미스트: 빅데이터로부터 발생하는 문제 중간자 역할로서 중재자
-   빅데이터 위기 요인 (3가지)
    -   사생활 침해 → 동의에서 책임으로
    -   책임원칙 훼손 → 결과 기반 책임 원칙 고수
    -   데이터 오용 → 알고리즈미스트 필요
-   빅데이터 기술
    -   하둡, 아파치 스파크, 스마트 팩토리, 아마존
-   비식별화 방법
    -   데이터 마스킹, 가명처리, 총계처리, 데이터값 삭제, 데이터 범주화
