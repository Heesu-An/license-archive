## **<분석 방법론>**

-   🚨 분석 방법론의 구성 요소 (4가지)
    -   **절차, 방법, 도구와 기법, 템플릿과 산출물**
-   🚨 분석 방법론 모델 (3가지)
    -   폭포수형 모델 (하향식): 이전 끝 후 다음 넘어감. **모든 단계에 피드백** 단계 존재.
    -   프로토타이핑 (상향식): What, How를 모르지만 일단 함. **시제품** 반복 과정. 어자일.
    -   나선형 (상향식): 사이즈를 점점 키워감. **복잡도 관리** 필요.
-   🚨🚨 분석 방법론 (3가지)
    -   KDD 분석 방법론: 데이터셋 선택(=추출) → 전처리 → 변환(데이터분할, 차원축소) → 데이터마이닝 → 결과 평가
    -   CRISP-DM 분석 방법론(4레벨, 6단계): (**업데데**이트**모델평가전**) 업무 이해 → 데이터 이해 → 데이터 준비 → 모델링 → 평가 → 전개
        -   3계층: 단계 → 태스크 → 스탭
        -   5단계🚨🚨🚨 빅데이터 분석 방법론(3계층, 5단계):

    -   비지니스 이해 및 범위 설정 → 프로젝트 정의 및 계획 수립 → 프로젝트 위험 계획 수립
    -   위험 대응 계획 수립: (**회전완수**) 회피, 전이, 완화, 수용으로 구분하여 위험 관리 계획서 작성
    -   SOW(프로젝트 범위 정의서), WBS(프로젝트 정의서)를 포함하여 위험 계획 수립
-   데이터 준비
    -   필요 데이터 정의 → 저장공간 수립 → 데이터 수집 및 정합성 점검
    -   데이터 정의서 작성
-   데이터 분석
    -   분석용 데이터 준비 → 텍스트 분석 → 탐색적 분석 (EDA=데이터시각화) → \*\*\*모델링(데이터 분할, 데이터 모델링 및 모델링 결과 보고서, 의사코드 수준의 상세한 알고리즘 설명서)\*\*\*→ 모델 평가 및 검증(모델 튜닝)
-   시스템 구현(옵션)
    -   시스템 설계 및 구현 → 시스템 테스트 및 운영
-   평가 및 전개
    -   모델 발전 계획 수립 → 프로젝트 평가 및 보고

## **<분석 과제 발굴-하향식 접근법(4단계)>**

-   탐색 → 정의 → 해결방안 → 타당성 검토

**1\. 문제 탐색 (What)**

-   비지니스 모델 기반 문제 탐색: (업제고규지) 업무, 제품, 고객, 규제 및 감사, 지원 인프라

-   관점: 거시적 관점, 경쟁자 확대, 시장 니즈 탐색(채널), 역량의 재해석
-   외부 사례 기반 문제 탐색
-   유즈케이스 정의

**2\. 문제 정의(How)**: 데이터 및 기법를 정의하기 위해 분석 문제로의 변환

**3\. 해결방안 탐색**

**4\. 타당성 검토**

-   **경제적 타당성: 비용대비 편익**
-   **데이터 타당성: 데이터 존재여부 분석역량**
-   **기술적 타당성:** 역량 확보 방안 사전 수립

## **<분석 과제 발굴-상향식 접근법 (4가지)>**

-   주로 비지도학습
-   디자인 사고: 상향식의 발산 + 하향식의 수렴. 상호 보완. 동적인 환경에서 최적의 의사결정 방식. 감정 이입
-   상관관계 분석, 연관 분석
-   프로토타이핑: 가설생성 → 디자인 실험(가설 실험) → 테스트 → 통찰 도출 및 가설 확인

-   🚨 분석 과제 관리 영역 (5가지)
    -   기존 영역 5가지 + 추가 관리 영역 5가지
    -   **데이터 크기, 데이터 복잡도, 속도, 분석 모델의 복잡도, 정확도 & 정밀도(Trade-off)**

-   프로젝트 관리 지식 체계 (10가지)
    -   (**이범통**이 **의자**에서 **시원**한 **조리품**을 먹었다) 이해관계자, 범위, 통합, 의사소통, 인적자원, 시간, 원가, 조달(아웃소싱), 리스크, 품질
